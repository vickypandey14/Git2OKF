# Security Policy

## Supported Versions

We actively maintain and provide security updates for the following versions of Git2OKF:

| Version | Supported |
| --- | --- |
| Development (Main) | Yes |
| < 0.2.0 | No |

We recommend always running the latest code from the `main` branch during active architectural developments.

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue. Instead, report it directly to the repository maintainers or owners.

We request that you include:
1. A detailed description of the vulnerability.
2. Steps, code, or commands required to reproduce the behavior (proof of concept).
3. The potential impact of the issue.

## Response Process

Once a vulnerability report is received, the maintainers will:
* Acknowledge receipt of the report within 48 hours.
* Work to confirm the issue and determine its impact.
* Develop a fix or mitigation strategy.
* Coordinate a release containing the resolution.
